## MISSAO 005: Saneamento do Contrato

### 1. RESUMO

Contrato `SocialCareContract` atualizado para usar DTOs tipados. Eliminado o anti-padrao de `Map<String, dynamic>` em `listPatients` e `Patient` domain model em `getPatient`/`getPatientByPersonId`. O "falsificador de JSON" (`_extractSummary`) foi substituido por `_toSummaryDTO` que retorna `PatientSummaryDTO` tipado.

### 2. ARQUIVOS CRIADOS

| Arquivo | Conteudo |
|---------|----------|
| `bff/shared/lib/src/infrastructure/dtos/patient_summary_dto.dart` | DTO leve para listagem (patientId, personId, firstName, lastName, fullName, primaryDiagnosis, memberCount) |
| `bff/shared/lib/src/infrastructure/dtos/patient_summary_dto.g.dart` | Codigo gerado (validado pelo build_runner) |

| Metodo | Antes | Depois |

### 4. IMPLEMENTADORES ATUALIZADOS

#### 4.1 `LocalSocialCareRepository` (bff/social_care_desktop)
- `listPatients()` — retorna `Result<List<PatientSummaryDTO>>`

- **ADICIONADO:** `_toSummaryDTO(CachedPatient)` → `PatientSummaryDTO` (tipado, mesma logica de extracao)
- `getPatient()` / `getPatientByPersonId()` — retornam `Result<PatientDTO>` via `PatientDTO.fromJson(json)`
- **ADICIONADO:** `updateCacheFromDTO(PatientDTO)` — persiste DTO no cache local (usado pelo OfflineFirstRepository)
- `updateCacheFromSummaries()` — agora aceita `List<PatientSummaryDTO>` em vez de `List<Map<String, dynamic>>`

#### 4.2 `SocialCareBffRemote` (bff/social_care_desktop)
- `listPatients()` — mapeia response JSON para `List<PatientSummaryDTO>` via `PatientSummaryDTO.fromJson`
- `getPatient()` / `getPatientByPersonId()` — retornam `Success(PatientDTO.fromJson(data))` em vez de `PatientMapper.fromJson(data)`

#### 4.3 `OfflineFirstRepository` (bff/social_care_desktop)
- `listPatients()` — type change (delegates ao local/remote que ja retornam DTO)
- `getPatient()` / `getPatientByPersonId()` — `onRemoteSuccess` agora usa `updateCacheFromDTO(dto)` em vez de `updateCache(patient)`

3. **Fluxo de dados resultante (limpo):**


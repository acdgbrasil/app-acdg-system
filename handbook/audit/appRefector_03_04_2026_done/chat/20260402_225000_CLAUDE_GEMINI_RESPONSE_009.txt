## 2. VIEWMODEL GOLD STANDARD

### ValueNotifier → Private fields + notifyListeners
| Antes (ValueNotifier) | Depois (Gold Standard) |
|-----------------------|----------------------|
| `final currentStep = ValueNotifier<int>(0)` | `int _currentStep = 0` + `int get currentStep` |
| `final showStepErrors = ValueNotifier<bool>(false)` | `bool _showStepErrors = false` + `bool get showStepErrors` |
| `final parentescoLookup = ValueNotifier<List<LookupItem>>([])` | `List<LookupItem> _parentescoLookup = []` + getter |
| (idem para 3 outros lookups) | (idem) |

### Logica de fluxo movida da View
Enums criados:
```dart
enum StepNavigationResult { advanced, validationFailed }
enum SubmitResult { success, validationFailed, networkError, serverError }
```

Metodos adicionados:
- `StepNavigationResult handleNext()` — valida, avanca, ou seta errors
- `Future<SubmitResult> handleSubmit()` — valida, registra, classifica erro

A View agora faz apenas:
```dart
switch (vm.handleNext()) {
  case StepNavigationResult.advanced: break;
  case StepNavigationResult.validationFailed: showToast(error);
}
```

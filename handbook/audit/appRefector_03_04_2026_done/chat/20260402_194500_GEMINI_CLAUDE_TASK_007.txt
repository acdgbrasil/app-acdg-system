## 1. OBJETIVO
Refatorar o `FamilyCompositionViewModel` para seguir o **Architectural Gold Standard**. Eliminar a gestão manual de loading, a reatividade redundante e o parsing de JSON na UI.

## 2. ARQUIVOS AFETADOS
- `packages/social_care/lib/src/ui/family_composition/viewModel/family_composition_view_model.dart`
- **AÇÃO OBRIGATÓRIA:** Renomear a pasta de `viewModel/` para `view_models/`.

## 3. INSTRUÇÕES TÉCNICAS (XP: RIGOR EXTREMO)
1. **Padrão Command:** 
   - Remova a flag `_isLoading` e o getter `isLoading`.
   - Substitua o método `loadPatient()` por `late final Command0<void> loadPatientCommand`.
   - A UI deve consumir `loadPatientCommand.running`.
2. **Fim do "JSON na UI":**
   - Remova o método `_extractMembers` que faz parsing manual de JSON (`json['key']`).
   - O ViewModel agora recebe o objeto `Patient` (Entidade de Domínio) a partir do UseCase (que agora retorna `Patient`, não mais `PatientDetailResult`).
   - Use um Mapper/Translator local para converter a Entidade `Patient` para o `FamilyMemberModel`.
3. **Lógica de Domínio:**
   - O cálculo do `ageProfile` não pertence ao ViewModel. Mova essa lógica para um serviço de domínio ou utilize a entidade `Patient` se ela já tiver essa capacidade. O VM apenas expõe o resultado.
4. **Reatividade:**
   - Garanta que não existam `ValueNotifier` dentro do ViewModel. Use campos privados e `notifyListeners()`.

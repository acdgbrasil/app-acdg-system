# MISSÃO 002: Transição para Riverpod (Injeção de Dependências)
# OBJETIVO: Eliminar o `AppProviders` e usar Contêineres Declarativos.

## 1. CONTEXTO
O `apps/acdg_system/lib/logic/di/app_providers.dart` está "inchado". Vamos começar a mover a injeção para o Riverpod.

## 2. TAREFAS
1. **Instalar Dependências:** Certifique-se de que `flutter_riverpod` e `riverpod_annotation` estão disponíveis.
2. **Auth Providers:** Crie o arquivo `apps/acdg_system/lib/logic/di/auth_providers.dart`.
3. **Implementar:**
   - Um `Provider` para `AuthRepository` (buscando do `AppDependencyManager` por enquanto, ou criando uma factory).
   - Um `Provider` para os UseCases (`LoginUseCase`, `LogoutUseCase`, `RestoreSessionUseCase`).
   - Um `ChangeNotifierProvider` para o `AuthViewModel`.
   
   *Dica estrutural:*
   ```dart
   final authViewModelProvider = ChangeNotifierProvider<AuthViewModel>((ref) {
     return AuthViewModel(
       authRepository: ref.watch(authRepositoryProvider),
       loginUseCase: ref.watch(loginUseCaseProvider),
       // ...
     );
   });
   ```

4. **Root Update:** No arquivo `apps/acdg_system/lib/main.dart` ou `root.dart`, envolva o app com `ProviderScope`.

## 3. REGRAS
- Não use `build_runner` ainda se não for estritamente necessário; use Providers manuais para acelerar esta fase de transição.
- O objetivo é que o `AppProviders` (MultiProvider) comece a encolher.

Aguardando sua estrutura de Providers.

Conecta Raros

Um ecossistema open source projetado para cuidado social de pacientes com doenças genéticas raras.

02

A Origem

De um TCC
à Produção.

O projeto nasceu como um Trabalho de Conclusão de Curso (TCC) sobre Edge Cloud Computing. A tese explorou como construir uma nuvem privada usando K3s, GitOps com FluxCD e Tailscale para conectividade mesh.

Mas a motivação real sempre foi a mesma: 13 milhões de brasileiros afetados por doenças raras, sem nenhum sistema centralizado de dados para ajudá-los.

A tese se tornou a fundação de um sistema real de produção, agora utilizado pela ACDG para gestão de assistência social a pacientes com doenças genéticas raras.

Da teoria
à prática.

Um TCC que virou infraestrutura real. Um servidor Xeon e Raspberry Pis que viraram uma edge cloud privada rodando Kubernetes em produção.

05

Decisões Arquiteturais

Cada escolha tem um motivo.

ADR 01

Por que Swift no backend?

Strict concurrency nativa com actors e Sendable. Type safety extrema que previne bugs em tempo de compilação. Performance próxima de C. Não é o mainstream, mas é a escolha certa para dados sensíveis de saúde onde cada byte de memória e cada thread importam.

ADR 02

Por que Flutter e não React?

Um único codebase para Web, Desktop e Mobile. Offline-first nativo com Drift para storage local. Sem dependência de browser para desktop — nada de Electron consumindo 500MB de RAM. Web compila para WASM com performance nativa.

ADR 03

Por que Edge Computing?

A ACDG opera em regiões com infraestrutura limitada. K3s rodando em Raspberry Pi permite processamento local com custo mínimo. Tailscale mesh VPN conecta todos os nós com segurança WireGuard, sem expor portas na internet.

ADR 04

Por que CQRS + Event Sourcing?

Histórico social é sagrado — nunca deletamos dados de pacientes. Event Sourcing mantém toda a história de mudanças como uma sequência de eventos imutáveis. CQRS separa leitura de escrita para performance, com Transactional Outbox para consistência.

ADR 05

Por que OpenAPI + AsyncAPI?

Contratos primeiro, código depois. Todos os serviços definem seus contratos ANTES da implementação. São aproximadamente 163 schemas YAML que cobrem todo o domínio de doenças raras — desde cadastro até triagem e proteção social.

06

Infraestrutura

Edge Cloud
Privada.

Uma nuvem privada construída com hardware acessível e software open source. Sem vendor lock-in, sem custos recorrentes de cloud.

VPS Gateway

Servidor na internet pública com Caddy para HTTPS automático. Ponto de entrada para todo o tráfego externo.

Tailscale Mesh

VPN mesh baseada em WireGuard que conecta todos os nós com segurança, sem expor portas na internet pública.

Xeon Master

Servidor Xeon como master node K3s. Control plane do cluster Kubernetes que orquestra todos os workloads.

Raspberry Pi Workers

Raspberry Pis como worker nodes na edge. Processamento distribuído com custo mínimo e consumo baixo de energia.

FluxCD GitOps

Deploy pull-based, não push. O cluster sincroniza automaticamente com o repositório Git. Rollback é um git revert.

Bitwarden Secrets

Zero segredos no Git. Bitwarden Secrets Manager injeta credenciais diretamente nos pods do Kubernetes.

Zitadel Self-Hosted

Identity Provider OIDC self-hosted. Controle total sobre autenticação e autorização, sem depender de SaaS externo.

07

Segurança

Privacidade como
princípio fundamental.

Dados de pacientes com doenças raras exigem o mais alto nível de proteção. Cada camada do sistema foi projetada com segurança desde o início.

LGPD Nativa

Conformidade com a Lei Geral de Proteção de Dados desde a concepção. Privacy by Design em todas as camadas.

Split-Token Pattern

Access token em memória Dart, refresh token em cookie HttpOnly. Nenhum token acessível por JavaScript no browser.

RBAC

Três papéis definidos: assistente social (CRUD), responsável legal (leitura) e administrador (leitura + gestão).

Audit Trail

Header X-Actor-Id obrigatório em toda mutação. Rastreabilidade completa de quem alterou o quê e quando.

Headers de Segurança

CSP com nonce, HSTS, X-Content-Type-Options: nosniff, X-Frame-Options: DENY, CORS strict mode.

CRU (sem Delete)

Create, Read, Update. Dados sociais nunca são deletados. Histórico completo preservado via Event Sourcing.

08

Open Source

Código aberto
para a
comunidade.

Todo o código do Conecta Raros é aberto no GitHub. A missão é que outras associações de doenças raras possam reutilizar a tecnologia para seus próprios pacientes.

Os contratos de API são definidos em OpenAPI e AsyncAPI, garantindo interoperabilidade entre diferentes instituições. Qualquer organização pode se integrar ao ecossistema seguindo os schemas compartilhados.
